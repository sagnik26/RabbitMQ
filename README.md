# RabbitMQ learn

| Level | Overview |
| ----- | -------- |
| [L1](L1/README.md) | **Direct exchange**, one queue, one consumer — smallest publish → bind → consume flow with a single routing key. |
| [L2](L2/README.md) | **Direct exchange**, two routing keys and two queues so different consumers receive different message types. |
| [L3](L3/README.md) | **Topic exchange**, dot-separated routing keys and pattern bindings (`order.*`, `payment.*`) with two notification-style consumers. |
